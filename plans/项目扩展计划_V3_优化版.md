### 阶段零：基础设施增强 (Week 0-1)

#### 0.1 缓存层引入 (P0)

**目标**: 引入Redis缓存，提升查询性能

**技术方案**:
```python
# backend/cache/
backend/
├── cache/
│   ├── __init__.py
│   ├── redis_client.py      # Redis客户端封装
│   ├── cache_manager.py     # 缓存管理器
│   ├── decorators.py        # 缓存装饰器
│   └── serializers.py       # 数据序列化
```

**缓存策略**:
| 数据类型 | 缓存时长 | 更新策略 |
|---------|----------|----------|
| 实时行情 | 5秒 | 主动更新 |
| K线数据 | 1小时 | LRU淘汰 |
| 股票列表 | 1天 | 定时刷新 |
| 技术指标 | 30分钟 | 主动更新 |
| 用户会话 | 24小时 | 滑动过期 |
| 选股结果 | 10分钟 | 主动更新 |

**缓存装饰器示例**:
```python
from backend.cache.decorators import cache_result

@cache_result(ttl=300, key_prefix="stock:quotes")
async def get_stock_quotes(symbol: str):
    # 自动缓存5分钟
    return await fetch_from_tdengine(symbol)
```

**指标消费者**:
```python
class IndicatorConsumer:
    """指标计算消费者"""

    async def on_bar_message(self, message: BarMessage):
        """接收K线消息，计算指标"""
        symbol = message.symbol
        bars = message.bars

        # 获取需要计算的指标
        indicators = await self.get_active_indicators(symbol)

        # 批量计算
        results = await self.calculate_batch(indicators, bars)

        # 存储到TDengine
        await self.store_indicators(symbol, results)

        # 发布消息
        await self.publish_indicators(symbol, results)
```

## 开发规范 (V3)

### Git工作流

```
main (生产环境)
  ↑
  └── develop (开发环境)
         ↑
         ├── feature/user-auth
         ├── feature/screener
         ├── feature/indicators
         └── fix/data-quality
```

### Commit规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型**:
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式（不影响逻辑）
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建或辅助工具变动

**示例**:
```
feat(screener): add custom expression filter

- Add support for user-defined expressions
- Implement expression parser and evaluator
- Add validation for expression syntax

Closes #123
```
